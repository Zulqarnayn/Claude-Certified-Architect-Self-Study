## Answer Guide

<details>
<summary><b>Q1 — Prompt rollback triggers</b></summary>

Roll back when:
- New prompt increases error/hallucination rate
- User satisfaction drops
- Metric regression appears in A/B test
- Critical bug introduced
- Upstream model or API change causes instability

</details>

<details>
<summary><b>Q2 — Documenting prompt changes</b></summary>

Use:
- Git commits with clear message
- Prompt change log file
- Review checklist
- Release notes for prompt versions
- Link prompt changes to evaluation metrics

</details>
