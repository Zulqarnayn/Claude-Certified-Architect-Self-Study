## Answer Guide

<details>
<summary><b>Q1 — Never personalize with sensitive data</b></summary>

Never personalize with:
- Passwords or authentication tokens
- Payment information
- Medical or legal secrets without consent
- Any data without explicit user permission

</details>

<details>
<summary><b>Q2 — Consistent personalization</b></summary>

- Use a small set of stable user preferences
- Keep core instruction consistent
- Personalize only tone, not core logic
- Validate personalized outputs with the same schema

</details>
